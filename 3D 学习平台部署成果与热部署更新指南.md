## 🛠️ 国内服务器专属：极速热更新方案

为了彻底解决国内服务器拉取 GitHub 代码容易卡死、超时的问题，我们在服务器上部署了**多节点备用加速下载 + 暴力无损解压覆盖**的极速更新脚本。

### 1. 极速热更新脚本内容 (`fast_update.sh`)

在你的服务器 `/var/www/3d-lms` 目录下运行以下命令，会自动创建极速热更新脚本并执行它：

```bash
# 1. 确保在项目目录
cd /var/www/3d-lms

# 2. 清理掉之前可能被错误覆盖进去的 SQLite 历史记录文件夹
rm -rf server/prisma/migrations

# 3. 写入终极完美版更新脚本
cat << 'EOF' > fast_update.sh
#!/bin/bash
echo "================================================"
echo "📥 [1/3] 开始尝试通过多个备用加速节点下载最新代码..."

URLS=(
    "https://gh-proxy.com/https://github.com/maobukeai/3d-personal-learning-platform/archive/refs/heads/main.zip"
    "https://ghproxy.net/https://github.com/maobukeai/3d-personal-learning-platform/archive/refs/heads/main.zip"
    "https://gh.ddlc.top/https://github.com/maobukeai/3d-personal-learning-platform/archive/refs/heads/main.zip"
    “https://cdn.ghproxy.net/https://github.com/maobukeai/3d-personal-learning-platform/archive/refs/heads/main.zip”
)

SUCCESS=0
for url in "${URLS[@]}"; do
    echo "-> 尝试连接节点..."
    if wget -T 10 -t 1 -q --show-progress "$url" -O update.zip; then
        echo "✅ 下载成功！"
        SUCCESS=1
        break
    else
        echo "❌ 该节点超时或失效，正在切换下一个节点..."
    fi
done

if [ $SUCCESS -eq 0 ]; then
    echo "🚨 抱歉，所有节点当前都无法连接，请稍后再试。"
    exit 1
fi

echo "📦 [2/3] 正在解压并静默覆盖旧代码..."
unzip -o -q update.zip

# 🌟 关键防护 1：在覆盖前，提前删掉源码里的 SQLite 历史记录，防止污染服务器！
rm -rf 3d-personal-learning-platform-main/server/prisma/migrations

cp -r 3d-personal-learning-platform-main/* ./
rm -rf 3d-personal-learning-platform-main update.zip

echo "🚀 [3/3] 代码覆盖完成！开始拉起部署流程..."
echo "================================================"

# 禁用 git pull，防止网络卡死
sed -i 's/git pull origin main/# git pull origin main/g' deploy.sh

# 🌟 关键防护 2：将严格的 migrate deploy 替换成更适合暴力更新的 db push
sed -i 's/npx prisma migrate deploy/npx prisma db push --accept-data-loss/g' deploy.sh

# 执行部署脚本
./deploy.sh
EOF

# 4. 赋予执行权限并执行更新
chmod +x fast_update.sh
./fast_update.sh
```

> [!TIP]
> **该脚本的精妙之处**：
> 1. **多路备用加速**：使用 4 个主流的国内 GitHub 代理加速节点，只要有一个节点通畅即可完成秒级下载。
> 2. **SQLite 防污染**：在解压覆盖前，提前将本地可能残留的 SQLite `prisma/migrations` 文件夹删掉，防止污染服务器端的 MySQL 结构。
> 3. **无缝平滑重载**：自动修改 `deploy.sh` 脚本，将国内不稳定的 `git pull` 屏蔽，并切换为效率更高、允许无损数据推包的 `prisma db push`，最后拉起编译和 PM2 平滑重载。

## 🚀 未来日常开发与更新实战指南

未来在开发迭代时，建议你按照以下三种常见场景进行维护：

### 💻 场景一：日常前端/后端代码更新（最常用）

如果你修改了 Vue 页面、CSS 样式、或者写了新的 Express 后端 API：

1. **本地开发电脑**：
   写完代码且本地测试正常后，将代码推送到你的 GitHub 仓库：
   ```bash
   git add .
   git commit -m "feat: 更新了某某 3D 功能组件"
   git push origin main
   ```
2. **京东云服务器**：
   登录服务器，进入目录，执行一键更新：
   
   ```bash
   cd /var/www/3d-lms
   ./fast_update.sh
   ```
   *（仅需 20 秒，你的最新代码就将以零停机（Zero-Downtime）的方式平滑上线！）*

---

### 🗄️ 场景二：数据库结构改变（使用 Prisma 增删字段）

如果你在本地开发时，修改了 `prisma/schema.prisma`（例如新增了数据库表或字段）：

1. **本地开发电脑**：
   在本地生成数据库迁移记录，并推送至 GitHub：
   ```bash
   cd server
   npx prisma migrate dev --name schema_update
   cd ..
   git add .
   git commit -m "db: 新增了某某字段"
   git push origin main
   ```
2. **京东云服务器**：
   直接运行一键更新：
   ```bash
   cd /var/www/3d-lms
   ./fast_update.sh
   ```
   *（`fast_update.sh` 会以极为安全的 MySQL 结构合并技术 `db push` 自动同步云端数据库，同时绝不会破坏已有的用户数据！）*

---

### 🔑 场景三：新增敏感环境变量（如新增 API 密钥）

如果你新增了敏感密钥（如支付宝正式 key，邮件客户端授权码等），因为 `.env` 文件被 Git 忽略了，你需要线上手动补充：

1. **本地开发电脑**：在本地的 `.env` 或 `server/.env` 中添加变量用于开发。
2. **京东云服务器**：
   ```bash
   # 编辑服务器后端环境变量文件
   nano /var/www/3d-lms/server/.env
   
   # 在文件末尾追加新变量：
   # NEW_API_KEY="your_value"
   #（按 Ctrl+O 保存，按 Enter 确认，按 Ctrl+X 退出）
   
   # 执行热更新以重载进程，使其读取新变量
   cd /var/www/3d-lms && ./fast_update.sh
   ```

## 🛡️ 常用运维监控排错小指令

在服务器上，你可以用这几组命令随时进行系统健康诊断：

```bash
# 1. 查看后端运行状态（内存与 CPU 占用）
pm2 list

# 2. 实时滚动查看后端 console 日志（排错利器 🐛）
pm2 logs 3d-lms-api

# 3. 监控 PM2 进程仪表盘
pm2 monit

# 4. 查看 Nginx 的实时网络连接日志
tail -f /var/log/nginx/3d-lms-access.log

# 5. 查看 Nginx 运行报错日志
tail -f /var/log/nginx/3d-lms-error.log
```
