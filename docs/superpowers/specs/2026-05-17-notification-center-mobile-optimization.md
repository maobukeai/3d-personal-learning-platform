# 消息中心移动端适配设计规格 (Mobile Optimization for Notification Center)

## 1. 背景与目标
当前消息中心页面 (`NotificationsView.vue`) 在手机端布局过于冗长，侧边栏占据了首屏大部分空间，通知内容需要大幅下滑才能看到。卡片样式在小屏幕上也不够精简。

目标：优化移动端布局，提升阅读效率和交互体验。

## 2. 设计方案

### 2.1 布局调整 (Layout)
- **侧边栏 (Sidebar)**：
    - 在手机端 (md 以下)，侧边栏不再作为垂直列表显示在顶部。
    - 改为**横向滚动的快捷 Tab 栏**，放置常用的通知分类 (全部、系统、团队等)。
    - “未读/全部”切换器移动到 Tab 栏上方或与搜索框并列。
- **搜索框**：
    - 在手机端进一步收窄高度，保持在 Tab 栏上方。
- **通知列表 (List)**：
    - 确保内容直接出现在首屏（Tab 栏下方）。
    - 列表容器在手机端去掉过大的边距 (`p-4` 减小)。

### 2.2 组件优化 (Components)
- **通知卡片 (Notification Card)**：
    - 圆角由 `rounded-3xl` 调整为 `rounded-2xl`，以适配手机屏幕。
    - 内边距由 `p-4 md:p-6` 调整为 `p-3.5`。
    - 图标大小由 `w-10 h-10` 略微缩小。
    - 标题与日期在极窄屏幕下自动折行。
- **顶部 Header**：
    - 缩小移动端高度。
    - 将“全部已读”和“清空”按钮在手机端简化为**图标按钮**或放入“更多”菜单中，以节省横向空间。

### 2.3 交互逻辑 (Interactions)
- 横向 Tab 点击后立即刷新列表内容（保持现有逻辑，仅改变视觉形态）。
- 搜索框保持实时过滤。

## 3. 成功标准
- 手机端打开页面，首屏至少能看到 2 条通知内容。
- 分类切换操作由“垂直寻找”变为“水平点选”，交互路径更短。
- 视觉风格与全局移动端设计语言（如 `MainLayout.vue`）保持一致。

## 4. 风险与权衡
- 横向滚动 Tab 的可发现性：通过渐变遮罩或滚动条提示用户。
- 分类过多时的处理：横向滚动是最佳方案，避免了下拉菜单的点击层级。
